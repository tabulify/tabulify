====== Bytle Db - Upsert (Update/Insert) a database ======


===== About =====
''upsert'' is an action of the [[start|database module]] that update a [[start|database]] or create it if it doesn't exist yet in the [[store|store]].


===== Syntax =====
<unit>
<code dos>
db database upsert --help 
</code>
<console>
Update or insert a database

Syntax:

	db database upsert [options|flags] <name>

where:

	Local
		* -u,--url <value>            The database url (if the database doesn't exist, this options is mandatory)
		* -l,--login <value>          The login (ie user)
		* -p,--password <value>       The user password
		* -pp,--passphrase <value>    A passphrase (master password) to encrypt the password
		* -d,--driver <value>         The jdbc driver (for a jdbc connection)
		* -s,--statement <value>      A inline statement to be executed after a connection

	Local
		* --store <value>             The path where the database information are stored

	Args:
		* name                        the database name

Example:  To upsert the database information about the database `name`:
    db database upsert --url jdbc:sqlite//%TMP%/db.db name
</console>
</unit>

===== Example =====
==== Modifying the database URL ====

  * Upsert a database
<unit>
<code dos>
db database upsert --url "jdbc:sqlite://%TMP%/db.db" mySqliteDB
</code>
<console>

</console>
</unit>
<unit>
  * With the [[info|info action]], we can see that the URL has changed
<code dos>
db database info mySqliteDB
</code>
<console>
-------------------------------
Name: mySqliteDB
URL: jdbc:sqlite://C:\Users\gerard\AppData\Local\Temp/db.db
Login: null
Password: null
Driver: null
Statement: null
-------------------------------
</console>
</unit>
