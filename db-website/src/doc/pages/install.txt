====== Bytle Db - Installation ======


===== About =====
This pages gives you all the steps needed to install successfully [[db|Bytle Db]].

===== Steps =====





==== Download ====
You need to download the zipped distribution file for your operating system.

''Bytle Db'' is distributed for different operation system.

==== Installation Directory ====


  * Create a directory called the [[bytle_db_home|Bytle Db Home]] where you will unzip the content of the installation file. 
<code dos>
:: Below c:\bytle-db is the Bytle Db Home. 
mkdir c:\bytle-db
</code>
  * Create an environment variable called ''BYTLE_DB_HOME'' with the path as value (In the documentation, we use it regularly to refer to resources such as SQL file)
<code dos>
setx BYTLE_DB_HOME=c:\bytle-db
</code>

==== Unzip ====
  * Unzip the downloaded file into the [[bytle_db_home|Bytle Db Home]]
<code dos>
unzip btyle-db-cli-1.1.0-SNAPSHOT-windows-x64.zip -d c:\bytle-db
</code>
==== Symlink ====

  * Optionally, create a symlink into your bin directory to be able to call [[db|Bytle Db]] from everywhere.
<code dos>
mklink c:\bytle-db\db.cmd %USERPROFILE%\bin\db.cmd
</code>
  * This local ''bin'' directory should be added to the ''PATH'' environment variable. 
<code dos>
setx PATH=%PATH%,%USERPROFILE%\bin
</code>

===== Installation validation =====

  * Open a console 
  * Call [[db|db]] with the ''help'' option
<unit>
<code dos>
db --help
:: or if you didn't have the directory in your PATH environment variable
:: BYTLE_DB_HOME\db.cmd --help
:: in our installation documentation
:: c:\bytle-db\db.cmd
</code>
  * The [[help|help]] should shows up.
<console>
A command line utility tool for every database

Syntax:

	db [options|flags] <command>

where:

	Command (is one of):
		* table                      operations on one or several tables
		* database                   operations on database
		* schema                     operations on a schema
		* query                      operations on a query
		* sample                     create and fill data into a sample schema
		* fkey                       operations on foreign key (ie on relation and constraints between tables)

	Local
		* -h,--help                  Print this help
		* --conf <path>              The path to the config file

To load data from a csv file you would type:
db table load TABLE_NAME.csv
</console>
</unit>
